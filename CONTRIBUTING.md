# 贡献指南

感谢你对 FiveM Studio 项目的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告 Bug

如果你发现了 bug，请：

1. 检查 [Issues](https://github.com/Pe0ny9-A/Fivem-Studio/issues) 中是否已经存在
2. 如果不存在，创建一个新的 Issue，使用 Bug Report 模板
3. 提供详细的重现步骤和环境信息

### 提出新功能

如果你有功能建议：

1. 检查 [Issues](https://github.com/Pe0ny9-A/Fivem-Studio/issues) 中是否已经存在
2. 如果不存在，创建一个新的 Issue，使用 Feature Request 模板
3. 详细描述功能的使用场景和预期效果

### 提交代码

1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request

## 代码规范

### 代码风格

- 使用 TypeScript 进行类型检查
- 遵循 ESLint 规则
- 使用 Prettier 格式化代码（如果配置了）
- 使用有意义的变量和函数名

### 提交信息

提交信息应该清晰描述更改内容：

- `feat: 添加新功能`
- `fix: 修复 bug`
- `docs: 更新文档`
- `style: 代码格式调整`
- `refactor: 代码重构`
- `test: 添加测试`
- `chore: 构建过程或辅助工具的变动`

## 开发环境设置

1. 克隆仓库
   ```bash
   git clone https://github.com/Pe0ny9-A/Fivem-Studio.git
   cd Fivem-Studio
   ```

2. 安装依赖
   ```bash
   npm install
   ```

3. 启动开发服务器
   ```bash
   npm run dev
   ```

4. 运行 lint
   ```bash
   npm run lint
   ```

## Pull Request 流程

1. 确保你的代码通过了所有测试
2. 更新相关文档
3. 确保提交信息清晰
4. 等待代码审查
5. 根据反馈进行修改

## 行为准则

- 尊重所有贡献者
- 接受建设性批评
- 专注于对项目最有利的事情
- 对其他社区成员表示同理心

感谢你的贡献！
